# Contributing to Hudhud

Thank you for your interest in contributing!

## Getting started

1. Fork the repository and create a branch from `main`.
2. Install dependencies: `bun install`
3. Start the dev server: `bun run dev`

## Making changes

- Follow the existing code style (enforced by Biome).
- Run `bun run lint` before submitting — CI will reject lint errors.
- Run `bun run typecheck` to catch type errors.
- Format with `bun run format` if needed.

## Submitting a pull request

- Keep pull requests focused on a single change.
- Describe **why** the change is needed in the PR description.
- Link any related issues.

## Reporting bugs

Use the [bug report template](.github/ISSUE_TEMPLATE/bug_report.md) to open an issue.
